主帖

一次随口提问，为什么会被模型永久当成你的深层兴趣？

Andrej Karpathy2026-03-26 00:05x.com/karpathy/status/2036836816654147718

Karpathy 说，当前各家 LLM 的个性化功能里，一个常见问题是：memory 很容易变成干扰项。你两个月前只是随口问过一次某个话题，模型却会在之后很长时间里，不停把它当成你的“深层兴趣”反复提起。

这种感觉不是自然的理解，而像是系统过度努力地表现自己很懂你。问题不在“有没有记住”，而在“记住了什么、怎样记、怎样再次提起”。

补充帖

这不像某一家实现的问题，更像更深层的训练偏向

Andrej Karpathy2026-03-26x.com/karpathy/status/2036841069636370467

Karpathy 补充说，他轮流使用各种 LLM，几乎都会遇到这个问题，所以这可能不是某一个产品实现得差，而是一个更深层的共性问题。

他的猜测是：在训练阶段，上下文窗口里的信息大多确实和任务相关，于是模型形成了一种强偏好——“只要上下文里给了东西，就应该尽量用上。” 到了实际使用时，memory feature 又把某些历史片段 RAG 回来，模型就会对这些被召回的记忆过拟合。

补充帖

很多 memory 可能只是 naive RAG

Andrej Karpathy2026-03-26x.com/karpathy/status/2036844441236103487

Karpathy 进一步猜测，问题未必只是“没有 decay”，更可能是：这些 memory 的实现方式本质上很像朴素版 RAG。你最终只能受制于 embedding 检索 top-k 恰好捞到了什么。

系统并没有真正跨时间整合、聚合、提炼“你是谁”，所以它很难分辨哪些只是短暂兴趣，哪些是持续偏好，哪些只是一次性的上下文噪音。这里的问题已经不是“检索错了”，而是人格建模过于粗糙。

评论支线

为什么模型总爱追问下一步？

Anubhaw Mathur × Andrej Karpathy2026-03-26x.com/AnubhawM/status/2036849257659940907

评论区有人把另一个很多人都烦的问题说穿了：为什么模型总爱在回答末尾追加一句——“接下来要不要我顺便继续做下一步？” 问题不在于不能提建议，而在于这种 follow-up 很多时候并不真的贴合用户当前意图，更像是在试图延长对话。

Karpathy 的回应非常直接：这大概率就是 engagement-maxxing。从 A/B test 看它可能效果很好，但它不像真实朋友会说的话，会显得 sleazy、weird。用户会觉得：一方面系统在想办法让你继续聊下去；另一方面它又抛出问题，让你不回答反而有点尴尬。

总结

这串讨论真正完成的三步推进

推进 1｜从检索问题，推进到人格建模问题

关键不只是 recall 准不准，而是系统能否区分一次性信息、长期偏好、历史噪音与稳定身份。

推进 2｜把记忆问题和产品动机问题接上

错误回捞记忆，再叠加不断追问的 follow-up，会让用户感受到的不是帮助，而是被 engagement 逻辑牵引。

推进 3｜把改进方向指向时间、频率与活跃项目

更像样的 memory system 至少要有时间衰减、频率强化、项目绑定，以及跨时间的聚合建模。

一句话总结：坏的个性化不会让人觉得“被理解”，只会让人觉得：系统正在用过时碎片和社交技巧，过度努力地讨好我。
